Everything you need to get started with Ravi

Overview

Ravi is an AI-powered pull request reviewer that analyzes your code changes alongside your database schema to provide intelligent feedback about performance impacts, schema changes, and potential issues.

By connecting to your database metadata (not actual data), Ravi can detect:

  • Database schema changes and migrations
  • Potential N+1 query problems
  • Missing database indexes
  • Table growth and performance impacts
Database Privacy

Ravi only accesses database metadata like table names and column types. We never read actual customer data from your tables.

Setup

Get started with Ravi in three simple steps:

Install GitHub App

Install the Ravi GitHub App on your organization or repositories. This allows Ravi to receive pull request events and post review comments.

Go to Dashboard

Configure Database Access

Create a read-only PostgreSQL user for Ravi to inspect your database structure. See the database configuration section below for detailed instructions.

Create Pull Requests

Once configured, Ravi will automatically analyze new pull requests and post review comments with insights about database impacts.

Database Configuration

To enable database analysis, you need to create a special read-only PostgreSQL user that Ravi can use to inspect your database structure and statistics.

Important

The read-only user will have access to database metadata (table names, column names, data types) but not to actual customer data.

Setup Steps

1. Download the SQL Script

Download our PostgreSQL setup script from your dashboard.

Go to Dashboard

2. Customize the Script

Open the downloaded file in a text editor and update these variables at the top.

\set db_name 'YOUR_DATABASE_NAME'
\set password 'YOUR_SECURE_PASSWORD'

3. Run the Script

Execute it with PostgreSQL superuser privileges (replace your_database with your actual database name):

psql -U postgres -d your_database -f setup_readonly_user.sql

4. Configure in Ravi

Go to your dashboard, click "Configure Database" for a repository. You can either provide a full connection string (DSN) or fill in the individual connection details:

Option 1: Connection String (DSN)

postgresql://prx_readonly:your_password@your_host:5432/your_database

Option 2: Individual Fields

  • Host: Your database server hostname
  • Port: 5432 (default PostgreSQL port)
  • Database: The name you set in step 2
  • Username: prx_readonly (or your custom name)
  • Password: The password you set in step 2

5. Test the Connection

Use the "Test Connection" button to verify everything works correctly.

What Ravi Can Access

The read-only user has access to:

  • information_schema - Table and column metadata
  • pg_catalog - PostgreSQL system catalogs
  • pg_stat_* views - Table statistics and row counts (estimates)
  • System functions for table sizes and index information

The read-only user cannot access:

  • Actual data rows in your application tables
  • Write operations (INSERT, UPDATE, DELETE)
  • Schema modifications (CREATE, ALTER, DROP)
  • User management operations

How It Works

Once configured, Ravi's workflow is automatic:

  1. PR Created: GitHub sends a webhook when a pull request is opened or updated
  2. Code Analysis: Ravi analyzes the code changes using AI
  3. Database Context: Ravi queries your database metadata to understand schema and performance
  4. AI Review: An AI model combines code with database context to identify issues
  5. Feedback Posted: Ravi posts review comments on your pull request
What Ravi Looks For

Ravi specializes in database-related performance issues: N+1 queries, missing indexes, missing filters, inefficient algorithms, schema changes, and data isolation problems.
