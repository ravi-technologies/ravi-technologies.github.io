Getting Started

How does Ravi understand my system without copying data?

Ravi ingests metadata only: table names, schemas, row counts, query plans, and repository context. It never pulls customer rows or secrets, so you get architectural insight without sharing production data.

What languages and databases are supported today?

We focus on teams running PostgreSQL or MySQL talking to back ends written in Ruby, Python, TypeScript/Node.js, or Go. Additional languages and datastores are on our roadmap, so reach out if you have a specific stack.

Will Ravi slow our PR workflow down?

No. Reviews arrive in parallel with your existing CI checks. Ravi skips style nits and only flags issues that will affect production behaviour, so engineers stay in flow while still getting architect-level feedback.

Security & Access

Can I control what context Ravi sees?

Yes. You decide which repositories and databases are connected. Access is read-only, scoped through service accounts, and can be revoked at any time.

How long does setup take?

Most teams finish onboarding in under an hour: install the GitHub app, point Ravi at your primary database (metadata-only), and select which PRs should run architectural checks.
