Pagination that degrades at scale (offset vs cursor)

Offset pagination is fine at 8K rows. At 5M rows, deep pages turn into slow queries. Ravi recommends cursor pagination only once the scale makes it necessary.

FAQ

Quick answers to the questions teams ask before trying Ravi. The full FAQ goes deeper on workflow, security, and privacy.

What is Ravi and how is it different from other code review tools?

Ravi brings system-level awareness to every pull request. Unlike other code reviewers that focus on syntax or basic static analysis, Ravi understands your production reality: request volumes, latencies, API contracts, and traffic patterns.

How long does setup take?

It should just be a few minutes. You install the Ravi GitHub App and select the repos. That's it.

Will Ravi slow our PR workflow down?

No. Reviews run in parallel with your existing CI and typically complete within 2-3 minutes. We focus on production-impacting issues, not style nitpicks.

My senior engineers review everything

Even senior reviewers can't simulate production scale on every PR. Let them focus on design while Ravi automates mechanical query and schema checks.

What data does Ravi access?

We read your code from GitHub PRs. We never touch actual customer data: no user emails, payment info, or business records.

Is my code stored or used to train AI models?

No. We don't store your code in plaintext, we build a vectorised embedding of the structure (AST) of the code. We don't train models on it, and we don't use it for anything except giving you feedback on that specific PR.

Can I control what Ravi has access to?

Yes. Choose which repositories to connect. Revoke access instantly by removing the GitHub App.

Can Ravi review database migrations?

Absolutely. Migration review analyzes schema changes against your codebase usage. We catch missing rollback strategies, operations that will lock tables, and column removals that will break deployed code.
